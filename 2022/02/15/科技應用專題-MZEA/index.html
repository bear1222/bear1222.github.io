簡述

在這堂專題課中，我與我的組員共同建立了一個智慧型英文學習系統，過程中，我對 php 及 sql 的語法更加熟悉，也對 css 有了基本的了解，讓我更加確定往資訊學群發展的方向。

心得

由近期背單字時遇到的困難，發想了此專題題目，藉由將單字隨機打亂，並自動記錄每天需背誦的單字，讓背單字的過程變得簡單且有效率。由於之前有上過一些 php 及 sql 的相關課程，因此我負責撰寫多數基本功能，在過程中，我不只更熟悉 php 及 sql 的語法，學到更多的是該如何做取捨。如同存取單字時，我們可以選擇讓任務模式（註一）能運行較快，但需耗費較多空間的方式，抑或是節省存取空間，但須花費較多時間讀取單字，這兩種方法各有利弊，沒有何者一定較好，因此我們必須從各方面考量，選擇較能接受的缺點。這次的專題中，我認為我們的分工很成功，兩人都能發揮各自的專長，讓我們的系統，除了一開始預定要達成的目標以外，還額外新增了許多計畫之外的功能。希望經過持續更新後，能夠真正發布上線。註一：任務模式是結合遺忘曲線及間隔式複習的複習方式，在新增該單字的當天，及之後 1/3/6/14 天時，會使用意思模式（顯示英文單字，使用者在心中默想其意思，再在螢幕上選擇答案是否正確的方式）測驗，而相隔 15 天時，會使用拼字模式（顯示中文意思，使用者在螢幕上拼出其對應的單字）進行測驗。
